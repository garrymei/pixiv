## TabBar 页面

### 1. 首页 (Home)
- **页面名称**: 首页 (作品与日常)
- **页面路径**: `/pages/home/index`
- **页面目标**: 留存用户，展示高质量的二次元 UGC 内容（Cosplay返图、同人图、漫展日常）。
- **核心模块**:
  - 顶部导航栏（Tab 切换：关注、推荐、最新）。
  - Banner 轮播图（展示官方置顶活动或优质内容）。
  - 瀑布流内容列表（展示帖子封面、标题、作者头像昵称、点赞数）。
- **输入数据**: Banner 列表数据、帖子分页列表数据 (支持下拉刷新 `page=1`，上拉加载 `page++`)。
- **输出行为**: 切换 Tab、点击帖子进入详情。
- **用户交互**:
  - 点击 Banner：跳转到对应活动详情或外部链接（H5）。
  - 点击帖子卡片：带 `id` 参数跳转至 [帖子详情页](#6-帖子详情页-post-detail)。
  - 卡片右下角点赞图标：未登录弹出登录框；已登录则爱心变红，数字 +1（乐观更新请求 API）。
- **空状态**:
  - 关注 Tab：显示插画 + “你还没有关注任何人，去推荐看看吧”。
  - 列表无数据：显示插画 + “暂无内容，快来发布第一篇吧”。
- **错误状态**: 网络超时显示“加载失败，点击重试”按钮。
- **是否属于 MVP**: 是

### 2. 发现/需求大厅 (Discovery/Demands)
- **页面名称**: 需求大厅
- **页面路径**: `/pages/discovery/index`
- **页面目标**: 撮合大湾区二次元创作者（摄影、模特、妆娘、后期）的合作。
- **核心模块**:
  - 顶部筛选栏（分类：全部/摄影/模特/化妆等；城市：全部/广州/深圳等；预算：全部/互免/有偿）。
  - 需求卡片列表（标题、类型标签、时间、地点、预算类型、发布者信息）。
- **输入数据**: 筛选参数（type, city, budget）、需求分页列表数据。
- **输出行为**: 发起合作申请。
- **用户交互**:
  - 点击筛选条件：重置列表并按新条件请求第一页数据。
  - 点击需求卡片：带 `id` 参数跳转至 [需求详情页](#9-需求详情页-demand-detail)。
  - 悬浮操作按钮 (FAB)：点击跳转至 [发布需求页](#8-发布需求页-create-demand)（需登录拦截）。
- **空状态**: 显示插画 + “暂无匹配需求，试试其他条件吧”。
- **错误状态**: 断网显示“获取需求失败，点击重试”。
- **是否属于 MVP**: 是

### 3. 发布 (Publish/Action)
- **页面名称**: 统一发布面板
- **页面路径**: `/pages/publish/index` (或作为一个全局半屏弹窗)
- **页面目标**: 提供清晰的创作入口分发。
- **核心模块**:
  - 半屏弹窗或独立页面。
  - 3 个大型圆形图标按钮：【发作品】、【发日常】、【发需求】。
- **输入数据**: 无。
- **输出行为**: 路由跳转。
- **用户交互**:
  - 页面加载即校验登录态，未登录则拦截至微信登录授权。
  - 点击【发作品】/【发日常】：带 `type` 参数跳转至 [发布作品/日常页](#7-发布作品日常页-create-post)。
  - 点击【发需求】：跳转至 [发布需求页](#8-发布需求页-create-demand)。
  - 点击底部【取消】或蒙层：关闭面板，停留在原页面。
- **空状态**: 无。
- **错误状态**: 无。
- **是否属于 MVP**: 是

### 4. 活动 (Events)
- **页面名称**: 官方活动与资讯
- **页面路径**: `/pages/events/index`
- **页面目标**: 集中展示大湾区漫展资讯和官方组织的招募活动。
- **核心模块**:
  - 顶部 Tab（资讯、官方活动）。
  - 列表（大图封面卡片，包含活动标题、时间、地点、状态角标[报名中/已结束]）。
- **输入数据**: 活动分页列表数据。
- **输出行为**: 浏览活动详情。
- **用户交互**:
  - 点击卡片：跳转至 [活动详情页](#10-活动详情页-event-detail)。
- **空状态**: 显示插画 + “近期暂无活动，敬请期待”。
- **错误状态**: 断网显示重试按钮。
- **是否属于 MVP**: 是

### 5. 我的 (Profile)
- **页面名称**: 个人中心
- **页面路径**: `/pages/profile/index`
- **页面目标**: 用户个人信息展示与个人资产管理。
- **核心模块**:
  - 顶部用户信息区（头像、昵称、一句话简介、编辑按钮）。
  - 数据统计区（关注数、粉丝数、获赞数 —— MVP阶段可能只做展示结构或部分实现）。
  - 核心功能入口区（我的发布、我的需求、我的活动）。
  - 底部设置区（关于我们、退出登录）。
- **输入数据**: 当前用户信息（`/api/v1/users/me`）、资产统计数据。
- **输出行为**: 进入各管理子页。
- **用户交互**:
  - 未登录态：头像显示默认图，昵称显示“点击登录”，点击触发登录流程。
  - 登录态点击“编辑资料”：跳转至 [编辑资料页](#14-编辑资料页-edit-profile)。
  - 点击功能入口栏：分别跳转至对应的“我的X”子页面。
  - 点击退出登录：清除本地 Token 和 UserInfo，重置为未登录态。
- **空状态**: 无（默认占位符）。
- **错误状态**: 获取用户信息失败时显示缓存数据或默认态。
- **是否属于 MVP**: 是

## 子页面

### 6. 帖子详情页 (Post Detail)
- **页面名称**: 帖子详情
- **页面路径**: `/pages/post/detail`
- **页面目标**: 消费帖子内容，参与互动（点赞、评论）。
- **核心模块**:
  - 顶部作者信息栏（头像、昵称、关注按钮）。
  - 轮播图/大图区（如果是多图展示 indicator）。
  - 正文区（标题、文字内容、标签、时间、地点）。
  - 评论列表区。
  - 底部吸底操作栏（输入框占位、点赞按钮及数量、评论按钮及数量）。
- **输入数据**: URL参数 `id`。调用 API 获取帖子详情数据和评论分页数据。
- **输出行为**: 点赞请求、提交评论请求。
- **用户交互**:
  - 左右滑动切换图片，点击图片进入微信原生全屏大图预览。
  - 点击关注/点赞：校验登录，切换状态（已关注/已点赞）。
  - 点击底部输入框：弹出真实键盘，输入内容后点击“发送”。发送成功后清空输入框，将新评论插入评论列表第一条，并提示“评论成功”。
  - 点击某条评论：弹出选项“回复 / 复制 / 举报”。
- **空状态**: 评论区无数据时显示“沙发空缺中，快来抢座吧”。
- **错误状态**: 帖子被删除或 ID 错误时显示“该内容已消失在异次元”，并提供返回上一页按钮。
- **是否属于 MVP**: 是

### 7. 发布作品/日常页 (Create Post)
- **页面名称**: 图文发布
- **页面路径**: `/pages/post/create`
- **页面目标**: 供用户上传图片并编写图文内容发布。
- **核心模块**:
  - 图片上传九宫格组件（限制最多 9 张）。
  - 标题输入框（单行，限制 30 字，日常类型可选填）。
  - 正文输入框（多行文本域，限制 1000 字）。
  - 标签选择器。
  - 底部“发布”大按钮。
- **输入数据**: 页面参数 `type` (work | daily)。
- **输出行为**: 上传图片到云端，提交表单数据到 API。
- **用户交互**:
  - 点击“+”调起微信选图相册。选图后显示缩略图，可点击右上角 `x` 删除。
  - 拖拽图片排序（如果技术允许，否则按选中顺序）。
  - 点击发布：显示“发布中...” Loading 蒙层。成功后 Toast 提示，随后 `wx.navigateBack` 并在上一页触发刷新；失败则 Toast 报错并保留当前草稿。
- **空状态**: 无。
- **错误状态**: 必填项（如未传图片）未填时点击发布，Toast 提示“请至少上传一张图片”。
- **是否属于 MVP**: 是

### 8. 发布需求页 (Create Demand)
- **页面名称**: 发布合作需求
- **页面路径**: `/pages/demand/create`
- **页面目标**: 结构化收集招募需求信息。
- **核心模块**:
  - 需求类型（Picker 选择：摄影/模特/妆娘等）。
  - 标题（单行）。
  - 详细描述（多行）。
  - 城市与地点（城市 Picker + 详细地址输入）。
  - 期望时间（日期时间 Picker）。
  - 预算设置（Radio 互免/有偿/待议，选有偿则展开金额输入框）。
  - 需要人数（步进器）。
- **输入数据**: 无。
- **输出行为**: 提交表单创建需求。
- **用户交互**:
  - 填写各项表单，利用原生 Picker 提升体验。
  - 点击提交：显示 Loading，成功后返回需求大厅并刷新列表。
- **空状态**: 无。
- **错误状态**: 漏填必填项时标红并 Toast 提示。
- **是否属于 MVP**: 是

### 9. 需求详情页 (Demand Detail)
- **页面名称**: 需求详情
- **页面路径**: `/pages/demand/detail`
- **页面目标**: 展示需求细则并提供申请入口。
- **核心模块**:
  - 需求状态条（招募中/已满员/已结束）。
  - 核心信息卡（类型、预算、时间、地点、人数进度 `1/3`）。
  - 描述详情。
  - 发布者信息。
  - 底部吸底操作栏（【立即申请】或【联系Ta】）。
- **输入数据**: 需求 `id`。
- **输出行为**: 发起申请 API。
- **用户交互**:
  - 浏览详情。
  - 若为本人发布的需求，底部按钮显示“管理需求”（第二阶段）。
  - 若为他人需求且招募中，点击【立即申请】：弹出模态框填写简单的申请附言，点击确认提交。成功后按钮变为“已申请”。
- **空状态**: 无。
- **错误状态**: 需求不存在提示。
- **是否属于 MVP**: 是

### 10. 活动详情页 (Event Detail)
- **页面名称**: 官方活动详情
- **页面路径**: `/pages/event/detail`
- **页面目标**: 展示漫展/活动图文说明，并引导报名。
- **核心模块**:
  - 顶部活动头图。
  - 活动标题与关键信息（时间、场馆）。
  - 活动详情长图文（富文本渲染）。
  - 底部吸底操作栏（分享按钮、报名按钮）。
- **输入数据**: 活动 `id`。
- **输出行为**: 唤起报名表单弹窗/页。
- **用户交互**:
  - 浏览富文本内容。
  - 点击“分享”唤起微信原生转发。
  - 点击“立即报名”：校验登录。弹出动态表单（姓名、电话、参展角色等），填写后提交。
  - 报名成功后，按钮置灰显示“已报名，待审核”。
- **空状态**: 无。
- **错误状态**: 富文本解析失败降级处理。
- **是否属于 MVP**: 是

### 11. 我的发布页 (My Posts)
- **页面名称**: 我的发布
- **页面路径**: `/pages/profile/my-posts`
- **页面目标**: 管理自己发布的作品和日常。
- **核心模块**: 双 Tab（作品、日常），瀑布流或列表结构。
- **输入数据**: API `/api/v1/users/me/posts`。
- **输出行为**: 查看、删除帖子。
- **用户交互**:
  - 点击卡片进入详情。
  - 长按卡片或点击卡片右上角 `...`：弹出“删除”选项，二次确认后调用删除 API，成功后从列表中移除该项。
- **空状态**: “你还没有发布过任何内容哦”。
- **是否属于 MVP**: 是

### 12. 我的活动页 (My Events)

- **页面路径**: `/pages/profile/my-events`

  - 点击卡片回到活动详情页。
- **空状态**: “你还没有参加过官方活动”。
- **是否属于 MVP**: 是

### 13. 我的需求页 (My Demands)
- **页面名称**: 我的需求
- **页面路径**: `/pages/profile/my-demands`
- **页面目标**: 管理自己发布的需求和发出的申请。
- **核心模块**: 双 Tab（我发布的、我参与的）。
- **输入数据**: 用户的需求列表/申请列表。
- **用户交互**:
  - 我发布的：点击可关闭招募/删除需求。
  - 我参与的：查看申请状态。

- **是否属于 MVP**: 是

### 14. 编辑资料页 (Edit Profile)
- **页面名称**: 完善个人信息
- **页面路径**: `/pages/profile/edit`
- **页面目标**: 修改系统中的用户画像。
- **核心模块**:
  - 头像上传区（点击替换）。
  - 表单组：昵称（Input）、性别（Picker）、常驻城市（Picker）、个性签名（Textarea）。
  - 保存按钮。
- **输入数据**: 现有 UserInfo 数据回显。
- **输出行为**: 提交 PUT 请求更新数据。
- **用户交互**:
  - 点击头像：调用微信选图和裁剪，上传获取 URL 后替换本地预览。
  - 修改文本后点击保存：Loading 提交，成功后更新全局 UserInfo 状态，返回上一页并 Toast“保存成功”。
- **空状态**: 无。
- **错误状态**: 昵称包含敏感词时，由后端返回错误，前端标红提示。
- **是否属于 MVP**: 是
