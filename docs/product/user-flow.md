# 粤次元君 (Yue Ci Yuan Jun) - 核心用户流程 (User Flow)

本文档定义了“粤次元君”小程序在 MVP 阶段的核心用户交互流程，确保开发人员对产品行为的理解一致。

## 1. 登录与注册流程 (Auth Flow)
- **触发时机**：用户首次打开小程序，或点击需要授权的操作（如点赞、评论、发布、报名）。
- **流程说明**：
  1. 用户进入小程序，默认可浏览首页、活动、需求等公开内容（游客模式）。
  2. 用户触发需要登录的操作（例如：点击“发布”按钮）。
  3. 弹出微信授权登录底部弹窗/跳转登录页。
  4. 用户同意授权，前端调用 `wx.login` 获取 code。
  5. 后端接收 code，通过微信接口换取 `openid`：
     - **新用户**：自动创建 `users` 表记录，生成系统 Token。
     - **老用户**：直接生成并返回系统 Token。
  6. 前端保存 Token，完成登录状态切换，继续执行刚才被中断的操作。

## 2. 帖子发布流程 (Post Creation Flow)
- **触发时机**：用户点击底部 TabBar 中间的“+”发布按钮。
- **流程说明**：
  1. 校验登录状态（未登录则引导登录）。
  2. 弹出选项：选择发布“作品”或“日常”。
  3. 进入发布编辑页：
     - 上传图片（最多9张），第一张默认作为 `cover_image`。
     - 填写标题（可选，通常作品需要标题）。
     - 填写正文内容。
     - 添加标签（可选）。
     - 选择位置（可选）。
  4. 用户点击“发布”。
  5. 前端上传图片到云存储/OSS，获取图片 URL 列表。
  6. 前端携带内容数据和图片 URL 调用发布 API。
  7. 后端将数据写入 `posts` 表和 `post_images` 表。
  8. 发布成功，提示用户并跳转回首页，刷新列表。

## 3. 浏览与互动流程 (Browse & Interact Flow)
- **触发时机**：用户在首页或帖子详情页浏览内容。
- **流程说明**：
  1. **浏览**：用户进入首页，按瀑布流展示作品/日常（支持下拉刷新、上拉加载更多）。
  2. **详情**：点击某张卡片，进入帖子详情页，展示大图轮播和图文详情。
  3. **点赞**：
     - 用户点击“爱心”图标。
     - 校验登录状态。
     - 前端立即点亮爱心并数量+1（乐观更新）。
     - 后端写入 `likes` 表记录，并更新 `posts.like_count`。
  4. **评论**：
     - 用户点击底部输入框或“评论”图标。
     - 校验登录状态。
     - 弹出键盘输入评论内容。
     - 提交后，后端写入 `comments` 表，更新 `posts.comment_count`。
     - 前端将新评论插入评论列表顶部。

## 4. 需求发布与接单流程 (Demand Flow)
- **触发时机**：用户在“需求”大厅寻找合作，或自己发布需求。
- **流程说明**：
  - **发布需求**：
    1. 点击需求大厅的“发布需求”按钮。
    2. 填写需求表单：类型（摄影/模特等）、标题、描述、预算（互免/有偿）、时间、人数等。
    3. 提交审核/直接发布，写入 `demands` 表。
  - **申请接单/参与**：
    1. 用户浏览需求列表，点击进入需求详情。
    2. 发现匹配的合作机会，点击“申请参与/接单”。
    3. 填写简单的留言或自我介绍（例如：“出原神刻晴，有档期”）。
    4. 提交后写入 `demand_applications` 表。
    5. （第二阶段：发布者收到通知并在个人中心确认合作）。

## 5. 活动报名流程 (Event Registration Flow)
- **触发时机**：用户在“活动”板块查看官方活动并报名。
- **流程说明**：
  1. 用户浏览官方活动列表（如漫展招募、同人展报名）。
  2. 进入活动详情，查看活动时间、地点、要求。
  3. 若活动状态为“报名中”且未满员，显示“立即报名”按钮。
  4. 点击“立即报名”，校验登录状态。
  5. 填写报名表单（根据不同活动动态生成的表单，如姓名、联系方式、CN/CN名等）。
  6. 提交报名，后端将表单数据 JSON 序列化存入 `event_registrations` 表。
  7. 提示报名成功，等待官方审核。
