# 粤次元君 - 阶段 5 接口异常与容错测试报告 V1

## 一、测试目标

验证 MVP 主链路在接口异常、空数据与空字段场景下是否具备清晰反馈、可重试能力、登录态恢复能力，以及是否会出现白屏或静默失败。

## 二、测试环境

- 测试日期：2026-04-02
- 后端复测地址：`http://127.0.0.1:3201`
- 鉴权方式：真实 token
- 执行方式：
  - 通过真实接口请求验证 401/403/404/400/空数组/null 字段场景
  - 结合前端请求层与页面异常分支代码审查，确认异常提示、重试与空态表现
  - 网络超时场景基于请求层实现做代码级验证

## 三、总体结论

- 共覆盖 7 类异常/容错场景。
- 通过：6 项
- 部分通过：1 项
- 阻塞：0 项
- 修复前曾存在“上传接口异常返回 500 internal error”问题，当前版本已降级为明确的 400 错误并可被前端识别。
- 当前前端已具备统一异常消息归一、401 自动刷新登录态重试、列表/详情页错误空态与重试入口。

## 五、详细记录

### 5.1 401 token 失效

- 接口验证：
  - 请求：`GET /posts/me`，携带非法 token
  - 返回：`401`，响应体为 `{"code":4002,"message":"invalid token","data":null}`
- 前端表现检查：
  - 请求层识别 `401` 或 `4002` 后，会清理本地登录态并重新触发 `bootstrapSession(true)`
  - 重试仍失败时，错误会继续向页面抛出，不会静默吞掉
  - 列表/详情页普遍采用 `loading -> error/empty -> retry` 结构，不会因为异常直接白屏
- 结论：通过

### 5.2 403 权限不足

- 接口验证：
  - 请求：`POST /events/2/register`
  - 返回：`403`，响应体为 `{"code":4003,"message":"not allow","data":null}`
- 前端表现检查：
  - 活动报名错误映射中已针对 `not allow` 输出“该活动为资讯内容，暂不支持报名”
  - 按钮点击后有 toast，不是无反馈失败
- 结论：通过

### 5.3 404 数据不存在

- 接口验证：
  - 请求：`POST /demands/999/apply`
  - 返回：`404`，响应体为 `{"code":4004,"message":"demand not found","data":null}`
- 前端表现检查：
  - 需求详情页加载失败时会展示 `EmptyState`
  - 操作型错误映射中已将 `demand not found` 转为“需求不存在或已下线”
  - 页面具备“重试”入口
- 结论：通过

### 5.4 500 服务异常

- 修复前验证：
  - 上传接口在无文件且未正确构造 multipart 请求时，曾返回 `500 / internal error`
  - 该问题说明服务端存在异常分支直接跌入全局 500 的情况
- 修复后验证：
  - `POST /uploads/image` 空文件请求当前返回 `400 / file required`
  - multipart 不完整请求返回 `400 / Multipart: Unexpected end of form`
- 前端表现检查：
  - 上传服务层会将异常包装为 `RequestError`
  - 发布页对上传失败有明确 toast 提示，并支持删除/重试失败图片
- 结论：通过

### 5.5 网络超时

- 实现验证：
  - 请求层统一设置 `timeout=10000`
  - 超时错误会归一成“网络请求超时，请稍后重试”
  - 其他网络错误会归一成“网络连接异常，请检查网络后重试”
- 限制说明：
  - 当前 CLI 环境未直接模拟真机断网与弱网超时
  - 本项结论来自请求层实现审查，不等同于真机弱网回归
- 结论：部分通过

### 5.6 空数组返回

- 接口验证：
  - 请求：`GET /events/me/registrations`
  - 返回：`200`，响应体为 `{"code":0,"message":"ok","data":{"list":[]}}`
- 前端表现检查：
  - 我的活动页会进入“暂无活动”空态
  - 不会出现 `list.map` 报错或白屏
- 结论：通过

### 5.7 null 字段返回

- 接口验证：
  - 请求：`GET /events/2`
  - 返回字段包含：`end_time=null`、`price=null`、`organizer=""`、`registration_deadline=null`
- 前端表现检查：
  - 服务层将 `null` 时间字段转换为 `undefined`
  - 页面显示默认状态文案、免费/待定等兜底内容
  - 未发现字段读取报错
- 结论：通过

## 七、结论与建议

- 401/403/404/空数组/null 字段这 5 类核心异常场景已具备可用的产品化反馈。
- 上传链路已补齐异常入参兜底，避免测试场景落入 500。
- 网络超时仍建议在真机弱网环境下补做一轮回归，重点确认 toast 时机与是否需要页面级重试按钮。
