# 粤次元君 - 阶段 5 高优先级问题修复报告 V1

## 一、修复目标

优先关闭阶段 5 测试中影响试用与上线判断的 P1 问题，并同步处理高频异常中的上传失败与请求层容错问题，确保修复项可复测。

## 二、修复范围

- 发布作品：必填校验、内容长度限制、重复提交防护
- 发布需求：必填校验、预算类型校验、重复提交防护
- 上传失败：无文件上传时的异常语义修复
- 请求层容错：统一超时配置与网络错误文案归一

## 三、修复清单

| 编号 | 修复项 | 根因 | 处理方案 | 结果 | 是否复测通过 |
| --- | --- | --- | --- | --- | --- |
| FIX-001 | 发布作品必须至少上传 1 张图片 | 后端 DTO 未限制 `images` 最小数量 | 为发帖 DTO 增加数组最小长度校验 | 已生效 | 是 |
| FIX-002 | 发布作品内容长度限制 | 后端 DTO 未限制 `content` 上限 | 将内容字段长度限制为 2000 字 | 已生效 | 是 |
| FIX-003 | 发布作品重复提交防护 | 服务层无幂等控制 | 基于作者与提交内容建立短时去重键，重复请求返回 409 | 已生效 | 是 |
| FIX-004 | 发布需求时间/地点/预算类型必填与合法性校验 | 需求 DTO 约束不足 | 补充 `event_time`、`location`、`budget_type` 强校验 | 已生效 | 是 |
| FIX-005 | 发布需求重复提交防护 | 服务层无幂等控制 | 基于作者与提交内容建立短时去重键，重复请求返回 409 | 已生效 | 是 |
| FIX-006 | 上传接口无文件请求返回内部错误 | 上传服务未校验空文件对象 | 对空文件请求返回 `400 / file required` | 已生效 | 是 |
| FIX-007 | 请求层缺少统一超时与网络错误文案 | `Taro.request` 未设置 timeout，错误归一不足 | 增加 `timeout=10000` 与超时/网络异常归一文案 | 已生效 | 是 |

## 四、复测记录

### 4.1 发布作品

| 场景 | 修复前 | 修复后 |
| --- | --- | --- |
| 空图片发布 | `201 created` | `400 images must contain at least 1 elements` |
| 超长内容发布 | `201 created` | `400 content must be shorter than or equal to 2000 characters` |
| 重复提交 | 两次均 `201 created` | 第二次返回 `409 duplicate submit` |

### 4.2 发布需求

| 场景 | 修复前 | 修复后 |
| --- | --- | --- |
| 缺少时间 | `201 created` | `400 event_time must be an integer number` |
| 缺少地点 | `201 created` | `400 location must be longer than or equal to 1 characters` |
| 非法预算类型 | `201 created` | `400 budget_type must be one of the following values: free, exchange, negotiable, fixed` |
| 重复提交 | 两次均 `201 created` | 第二次返回 `409 duplicate submit` |

### 4.3 上传与请求层容错

| 场景 | 修复前 | 修复后 |
| --- | --- | --- |
| `/uploads/image` 无文件请求 | `500 internal error` | `400 file required` |
| 网络超时 | 无统一 timeout 配置 | 统一 `timeout=10000`，并归一为超时提示 |
