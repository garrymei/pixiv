# 粤次元君 - 种子用户分层与邀请计划 V1

## 一、目标

冷启动阶段不能泛泛拉新，而是要优先邀请最可能带来内容、活动参与和传播的人。

本计划的目标是：

- 明确首批优先邀请哪些人
- 定义每类种子用户的价值
- 区分“谁负责发内容、谁负责参与活动、谁负责发需求、谁负责传播”
- 给出首批邀请策略建议

## 二、分层原则

- 优先邀请能产内容的人，而不是只看人数
- 优先邀请会参与、会互动的人，而不是只注册不动的人
- 优先邀请能带动二次传播的人，而不是单次使用者
- 官方账号和合作账号必须先到位，承担冷启动稳定输出

## 三、种子用户分层

### A 类：会发内容的人

典型人群：

- Coser
- 摄影师
- 妆造从业者 / 爱好者
- 有稳定出片习惯的内容型用户

### B 类：会参与活动的人

典型人群：

- 同城漫展常客
- 线下活动参与度高的同好
- 喜欢报名打卡、喜欢跟队参加活动的人

### C 类：会转发扩散的人

典型人群：

- 圈内活跃群友
- 小社团 / 同好群组织者
- 微博 / 小红书 / 朋友圈爱分享的人

### D 类：官方账号 / 合作账号

典型人群：

- 官方运营账号
- 合作摄影工作室
- 合作社团
- 合作活动组织者

## 四、每类用户的作用与目标动作

| 类别 | 核心作用 | 重点目标动作 |
| --- | --- | --- |
| A 类 | 负责发内容，撑起首页和发现页 | 发帖子、发返图、发招募、发布需求 |
| B 类 | 负责参与活动，形成热度和报名数据 | 报名活动、参与需求、互动评论 |
| C 类 | 负责传播和带人进场 | 转发活动、分享帖子、拉新扩散 |
| D 类 | 负责稳定供给和平台背书 | 发官方活动、发官方资讯、发品牌内容、发示范帖 |

## 五、分层目标拆解

### 5.1 A 类用户

目标：

- 首批就能产出正片、日常、返图
- 愿意发摄影招募、模特招募、妆造需求

建议动作：

- 发内容
- 发布需求
- 参与官方活动后产出返图

价值：

- 决定首页是不是空的
- 决定发现页标签是不是丰富的

### 5.2 B 类用户

目标：

- 让官方活动有人报名
- 让需求模块有人参与
- 让社区不是只有内容、没有互动

建议动作：

- 报名活动
- 参与需求
- 留下评论、点赞、互动

价值：

- 决定活动页和需求页是不是像“有活人”的平台

### 5.3 C 类用户

目标：

- 把优质内容和活动带到外部圈层
- 帮平台获得第一批真实流量入口

建议动作：

- 转发官方活动
- 转发品牌 / IP 内容
- 分享优质帖子或活动资讯

价值：

- 决定平台冷启动初期是否能突破“只靠官方自己发”

### 5.4 D 类用户

目标：

- 保证稳定供给
- 在用户内容不足时扛住首页和活动页更新

建议动作：

- 发布官方活动
- 发布活动资讯
- 发布示范帖
- 发布品牌 / IP 内容

价值：

- 决定平台前期是否有运营感、秩序感和可信度

## 六、首批邀请优先级

### P0：第一批必须先找

- 3~5 位 Coser
- 2~3 位摄影师
- 1~2 位妆造相关用户
- 1 个官方账号
- 1~2 个合作账号

### P1：第二批建议补

- 活动常参与用户
- 社团组织者
- 喜欢扩列和拉群的人

### P2：后续逐步扩大

- 普通围观型用户
- 单次活动参与者
- 偶发内容用户

## 七、邀请策略建议

### 7.1 A 类邀请策略

建议方式：

- 定向私聊邀请
- 以“首批内容共建”而非“泛泛试用”方式邀请
- 明确告诉对方：需要帮忙发 1~3 条内容

推荐话术方向：

- 平台刚起步，希望邀请首批内容型同好一起共建
- 可以优先展示作品、返图或招募内容
- 会优先给到首页展示或官方推荐位

### 7.2 B 类邀请策略

建议方式：

- 用“活动”或“同行招募”拉人，而不是单纯拉注册
- 让他们先通过活动进入平台，再沉淀为用户

推荐动作：

- 先邀请报名官方活动
- 再引导关注、评论、报名需求

### 7.3 C 类邀请策略

建议方式：

- 给明确可转发的内容，而不是只给平台链接
- 优先提供：
  - 活动海报
  - 活动资讯
  - 品牌 / IP 主题内容

推荐动作：

- 帮转活动
- 帮转主题挑战
- 帮带首批同城用户进来

### 7.4 D 类邀请策略

建议方式：

- 先确定官方账号更新机制
- 再筛选合作摄影工作室、社团、活动组织方

推荐动作：

- 固定发活动资讯
- 固定发活动预热
- 固定发示范帖

## 八、建议首批数量结构

| 类别 | 建议首批人数 / 账号数 | 目标 |
| --- | --- | --- |
| A 类 | 6~10 人 | 解决首页内容供给 |
| B 类 | 8~15 人 | 解决活动参与与基础互动 |
| C 类 | 5~8 人 | 解决冷启动扩散 |
| D 类 | 2~4 个账号 | 解决稳定更新和平台背书 |

## 九、冷启动阶段建议目标动作

### 第 1 阶段：先把内容立起来

重点依赖：

- A 类
- D 类

目标：

- 首页不空
- 活动页不空
- 需求页有真实内容

### 第 2 阶段：把参与感做出来

重点依赖：

- B 类
- A 类

目标：

- 官方活动有人报名
- 需求有人参与
- 帖子下有人互动

### 第 3 阶段：把扩散做出来

重点依赖：

- C 类
- D 类

目标：

- 把平台内容扩散到外部同好群与社交渠道

## 十、执行建议

推荐按以下顺序做种子用户邀请：

1. 先锁定 D 类账号，保证官方稳定更新
2. 再定向邀请 A 类内容用户，保证首页内容供给
3. 再补 B 类参与型用户，保证活动和需求不空转
4. 最后组织 C 类扩散型用户，做第一轮外部传播

## 十一、上线前检查 Checklist

- [ ] 已明确 A / B / C / D 四类种子用户
- [ ] 已确定首批优先邀请对象名单
- [ ] A 类用户能承担发内容与发需求动作
- [ ] B 类用户能承担活动报名与需求参与动作
- [ ] C 类用户能承担分享传播动作
- [ ] D 类账号能稳定更新资讯、活动和示范帖
- [ ] 首批邀请不是泛泛拉新，而是围绕目标动作组织

## 十二、可直接丢给 Trae 的 Prompt

要求：

1. 对首批用户进行分层
2. 说明每类用户的作用：发内容、参加活动、发布需求、帮助传播
3. 给出首批邀请策略建议
4. 输出 `seed-user-plan-v1.md`
