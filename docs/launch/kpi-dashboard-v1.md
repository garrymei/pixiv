# 粤次元君 - 冷启动阶段核心运营指标定义 V1

## 一、目标

冷启动阶段不能只凭感觉判断产品好不好用，必须先定义一版能持续追踪的核心指标。

本指标文档的目标是：

- 让团队知道现在到底该看什么
- 让“增长、内容、活动、需求、留存”都有明确口径
- 方便后续做轻量运营看板

## 二、使用原则

- 第一版只看核心指标，不追求过度复杂
- 优先定义能和当前产品三主线直接相关的指标
- 每个指标都要有定义、统计口径、作用
- 不使用泛泛的“活跃度很好”“用户挺多”这种模糊表达

## 三、指标分组

本阶段建议分为 5 组：

- 流量
- 内容
- 活动
- 需求
- 留存

## 四、流量指标

### 4.1 日访问人数

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 日访问人数 |
| 定义 | 当天至少进入过一次小程序的去重用户数 |
| 统计口径 | 以用户 ID 去重；无登录态时可用设备或会话标识临时辅助 |
| 作用 | 判断平台每天是否真的有人进来，而不是只有内容没人看 |

### 4.2 首页访问量

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 首页访问量 |
| 定义 | 当天首页被打开的总次数 |
| 统计口径 | 每次进入首页计 1 次，不去重 |
| 作用 | 判断首页是否是最主要流量入口，以及首页承接强不强 |

### 4.3 活动页访问量

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 活动页访问量 |
| 定义 | 当天活动页被打开的总次数 |
| 统计口径 | 每次进入活动页计 1 次，不去重 |
| 作用 | 判断活动内容是否能吸引用户继续浏览，而不是只停留在首页 |

## 五、内容指标

### 5.1 发帖人数

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 发帖人数 |
| 定义 | 在统计周期内至少成功发布 1 条帖子的人数 |
| 统计口径 | 以用户 ID 去重；只统计发布成功的帖子作者 |
| 作用 | 判断平台是否真的开始出现内容生产者 |

### 5.2 发帖数

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 发帖数 |
| 定义 | 在统计周期内成功发布的帖子数量 |
| 统计口径 | 仅统计状态为成功发布的帖子，不含草稿和失败提交 |
| 作用 | 判断内容池是否在增长，首页是否能持续更新 |

### 5.3 评论数

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 评论数 |
| 定义 | 在统计周期内新产生的评论总数 |
| 统计口径 | 统计评论成功写入的数据条数 |
| 作用 | 判断内容是否只是被看，还是开始产生互动 |

### 5.4 点赞数

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 点赞数 |
| 定义 | 在统计周期内新增的点赞总数 |
| 统计口径 | 统计成功点赞事件，不含取消点赞 |
| 作用 | 反映内容轻互动程度，适合早期观察“用户愿不愿意动手” |

## 六、活动指标

### 6.1 活动详情访问数

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 活动详情访问数 |
| 定义 | 活动详情页被打开的总次数 |
| 统计口径 | 每次进入活动详情页计 1 次，不去重 |
| 作用 | 判断活动资讯和活动卡片是否真的能把用户带到详情页 |

### 6.2 报名人数

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 报名人数 |
| 定义 | 在统计周期内成功报名官方活动的去重用户数 |
| 统计口径 | 以用户 ID 去重；只算报名成功 |
| 作用 | 判断官方活动是否有真实吸引力和参与度 |

### 6.3 报名转化率

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 报名转化率 |
| 定义 | 看到活动详情的用户中，最终完成报名的比例 |
| 统计口径 | 报名成功人数 ÷ 活动详情访问去重人数 |
| 作用 | 判断活动详情页是否讲清楚、报名动作是否顺畅 |

## 七、需求指标

### 7.1 需求发布数

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 需求发布数 |
| 定义 | 在统计周期内成功发布的需求数量 |
| 统计口径 | 只统计发布成功的需求记录 |
| 作用 | 判断需求模块是不是被真正使用，而不是只有浏览没有发布 |

### 7.2 需求参与数

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 需求参与数 |
| 定义 | 在统计周期内成功参与需求的次数或人数 |
| 统计口径 | 第一版建议先看成功参与次数；后续可再加参与人数去重 |
| 作用 | 判断需求是否真的形成协作，而不是只有发布没有响应 |

### 7.3 参与转化率

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 参与转化率 |
| 定义 | 进入需求详情的用户中，最终提交参与的比例 |
| 统计口径 | 参与成功人数或次数 ÷ 需求详情访问去重人数 |
| 作用 | 判断需求详情表达是否清楚、参与门槛是否过高 |

## 八、留存指标

### 8.1 次日留存

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 次日留存 |
| 定义 | 第一天访问过的用户，在第二天再次访问的比例 |
| 统计口径 | 第 D 天访问用户中，第 D+1 天再次访问的去重比例 |
| 作用 | 判断产品是否能让人第二天愿意回来 |

### 8.2 7 日留存

| 项目 | 内容 |
| --- | --- |
| 指标名称 | 7 日留存 |
| 定义 | 第一天访问过的用户，在第 7 天再次访问的比例 |
| 统计口径 | 第 D 天访问用户中，第 D+7 天再次访问的去重比例 |
| 作用 | 判断产品是否具备初步持续使用价值 |

说明：

- 7 日留存在冷启动前期可能样本较少，可先预留定义，后续再正式纳入看板

## 九、推荐第一版运营看板结构

建议首页看板按如下顺序展示：

### 今日概览

- 日访问人数
- 首页访问量
- 活动页访问量

### 内容

- 发帖人数
- 发帖数
- 评论数
- 点赞数

### 活动

- 活动详情访问数
- 报名人数
- 报名转化率

### 需求

- 需求发布数
- 需求参与数
- 参与转化率

### 留存

- 次日留存
- 7 日留存

## 十、指标使用建议

### 10.1 冷启动早期最值得先看的指标

优先建议每天盯：

- 日访问人数
- 首页访问量
- 发帖人数
- 发帖数
- 活动详情访问数
- 报名人数
- 需求参与数
- 次日留存

### 10.2 不建议过早过度关注

- 非常复杂的用户分群
- 长期价值模型
- 花式转化漏斗

当前阶段最关键的是先判断：

- 有没人来
- 有没人发
- 有没人报
- 有没人参与
- 来过的人第二天会不会回来
