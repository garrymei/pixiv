# 粤次元君 - V1.1 迭代候选范围建议

## 一、目标

V1.1 不是重新做一个新产品，而是在当前 MVP 基础上，继续围绕“内容、活动、需求”三主线增强体验。

本建议的目标是：

- 圈住最可能进入 V1.1 的功能
- 避免范围失控
- 明确哪些适合进入 V1.1，哪些不建议现在做

## 二、V1.1 规划原则

- 优先增强现有主链路
- 优先做能提升留存、互动、转化的功能
- 不做会显著拉大复杂度的重功能
- 不轻易新增与当前定位无关的大方向

## 三、当前 MVP 三主线

V1.1 仍应围绕以下三条主线推进：

- 内容互动增强
- 活动体验增强
- 需求协作增强

## 四、适合进入 V1.1 的候选功能

### 4.1 内容互动增强

#### 收藏

建议结论：

- 适合进入 V1.1

原因：

- 用户看到优质内容后需要一个“先留住”的轻互动动作
- 比复杂社交和复杂推荐更轻量，也更容易提升回访

#### 关注

建议结论：

- 适合进入 V1.1

原因：

- 有助于把内容消费从“看单条内容”升级到“关注某类人”
- 对 Coser、摄影师等内容型用户价值较高

#### 标签筛选优化

建议结论：

- 适合进入 V1.1

原因：

- 当前内容量一旦上来，标签筛选会直接影响浏览效率
- 属于内容主线的自然增强项

#### 简单搜索入口

建议结论：

- 谨慎进入 V1.1，可做轻量版本

原因：

- 冷启动阶段内容量还不大，复杂搜索收益有限
- 但一个轻量搜索入口有助于找帖子 / 活动 / 需求

建议方式：

- 先做简单关键词搜索入口
- 不做复杂排序、联想词、搜索推荐

## 五、活动体验增强

### 5.1 更多官方活动展示能力

建议结论：

- 适合进入 V1.1

原因：

- 当前活动页是平台冷启动的重要抓手
- 官方活动展示更丰富，能提升活动页存在感

建议增强方向：

- 官方活动卡片层次优化
- 活动预热、回顾、返图联动
- Banner 与活动详情联动更清楚

### 5.2 Banner 管理优化

建议结论：

- 适合进入 V1.1

原因：

- 冷启动阶段首页 Banner 是重要运营位
- Banner 可控性直接影响首页更新效率

### 5.3 举报入口

建议结论：

- 适合进入 V1.1

原因：

- 这是 UGC 产品的基本治理能力
- 比做复杂审核系统更应该优先进入下一轮

说明：

- 举报入口虽然不属于活动体验本身，但作为平台治理基础，适合并入 V1.1

## 六、需求协作增强

### 6.1 我的参与页优化

建议结论：

- 适合进入 V1.1

原因：

- 当前“我的参与”是需求协作闭环的重要结果页
- 优化后更能体现用户参与感和回访价值

建议增强方向：

- 状态更清楚
- 时间和参与记录更清楚
- 已参与内容更容易回看

### 6.2 需求展示优化

建议结论：

- 可与“我的参与页优化”一起进入 V1.1

原因：

- 需求模块对冷启动期很关键
- 用户是否愿意参与，和详情表达、状态展示强相关

## 七、不建议现在做的方向

### 7.1 不建议立刻进入 V1.1 的功能

| 功能方向 | 当前建议 |
| --- | --- |
| 复杂聊天系统 | 不建议 |
| 电商商城 | 不建议 |
| 重会员体系 | 不建议 |
| 泛社交关系链 | 不建议 |
| 复杂推荐算法 | 不建议 |
| 重后台系统 | 不建议 |

原因：

- 不直接增强当前三主线
- 开发成本高
- 容易让项目跑偏

## 八、推荐的 V1.1 候选清单

### P0 候选

- 收藏
- 关注
- 标签筛选优化
- 我的参与页优化
- 举报入口

### P1 候选

- Banner 管理优化
- 更多官方活动展示能力
- 轻量搜索入口

## 九、推荐不进入 V1.1 的项

- 复杂聊天系统
- 复杂搜索系统
- 电商与付费会员
- 大而全后台改造
- 泛社交化改造

## 十、建议的范围边界

### V1.1 应该继续聚焦

- 内容更好看、更好找、更值得回看
- 活动更容易看懂、更容易报名
- 需求更容易理解、更容易参与

### V1.1 不应该扩张到

- 另一个产品方向
- 另一个商业模式
- 明显超出 MVP 能力圈的大系统
