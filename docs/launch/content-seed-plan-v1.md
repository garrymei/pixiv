# 粤次元君 - 阶段 6 冷启动内容池规划 V1

## 一、目标

阶段 6 的重点不是单纯“功能可用”，而是让用户第一次进入小程序时，不会看到像空站、半成品站或只有测试数据的页面。

本规划聚焦冷启动期的内容池结构设计，不要求由 Trae 直接产出全部内容本身，但要把：

- 哪些内容必须先准备
- 每类建议准备多少
- 这些内容分别服务哪些页面和模块
- 上线最低保底量是多少

全部整理清楚。

## 二、规划原则

- 首页优先，先保证第一眼不空
- 活动页与需求页必须有连续可浏览内容，避免只有 1~2 条孤立数据
- 内容类型要混合，避免全是正片或全是招募
- 官方内容和用户内容都要有，既体现社区感，也体现运营感
- 冷启动阶段先追求“看起来有活跃度”，再追求“内容绝对丰富”

## 三、页面与模块映射

本次内容池规划主要服务以下页面：

| 页面 | 模块 | 需要承接的内容 |
| --- | --- | --- |
| 首页 | 推荐流 / 正片 / 日常 / 招募 | 帖子、需求、Banner |
| 发现页 | 推荐内容 / 标签 | 帖子内容与标签分布 |
| 活动页 | 活动资讯 / 官方活动 | 活动资讯、官方活动 |
| 需求详情与需求相关入口 | 需求列表承接 | 摄影 / 模特 / 妆造 / 毛发 / 道具需求 |
| 我的页相关 | 我的发布 / 我的需求 / 我的活动 / 我的参与 | 需要基础归属数据，但不作为本次规划主产出 |

## 四、冷启动内容池总量建议

### 4.1 建议总量

| 内容类别 | 建议数量 | 冷启动作用 |
| --- | --- | --- |
| 正片作品 | 8~15 条 | 支撑首页“正片”、推荐流、发现页内容密度 |
| 日常分享 | 8~15 条 | 支撑首页“日常”、提高活人感与社区氛围 |
| 活动返图 | 3~5 条 | 提高内容真实度，连接活动页与帖子内容 |
| 摄影招募 | 2~3 条 | 支撑需求页与首页招募流 |
| 模特招募 | 2~3 条 | 提升招募类型丰富度 |
| 妆造 / 毛发 / 道具需求 | 2~3 条 | 补足二次元垂类需求场景 |
| 活动资讯 | 5~10 条 | 保证活动页资讯 tab 不空 |
| 官方活动 | 1~3 条 | 保证活动页官方 tab 与首页 Banner 有承接 |

### 4.2 推荐冷启动保底量

如果阶段 6 上线前资源有限，建议至少达到以下保底量：

- 正片作品：8 条
- 日常分享：8 条
- 活动返图：3 条
- 摄影招募：2 条
- 模特招募：2 条
- 妆造 / 毛发 / 道具需求：2 条
- 活动资讯：5 条
- 官方活动：1 条

这套保底量的目标是：

- 首页看起来不是空的
- 活动页有连续浏览空间
- 需求页至少有 3 类不同需求

## 五、首页帖子内容规划

### 5.1 首页帖子池构成

首页建议以三层结构组成：

| 内容类型 | 建议数量 | 主要用于 |
| --- | --- | --- |
| 正片作品 | 8~15 条 | 首页推荐、首页“正片”、发现页推荐 |
| 日常分享 | 8~15 条 | 首页推荐、首页“日常”、发现页推荐 |
| 活动返图 | 3~5 条 | 首页推荐、活动氛围补充、活动详情外延内容 |

### 5.2 各模块承接关系

| 模块 | 内容来源 | 说明 |
| --- | --- | --- |
| 首页推荐流 | 正片 + 日常 + 活动返图混排 | 保证视觉层次和内容多样性 |
| 首页“正片”tab | 正片作品 | 最好至少一次可滚动看到 6~8 条 |
| 首页“日常”tab | 日常分享 | 避免只有精修图，体现真实社区使用感 |
| 发现页推荐 | 正片 + 日常 | 用于标签、搜索、内容筛选承接 |

### 5.3 首页最低展示目标

建议首页首次进入时至少能看到：

- Banner：1~2 条
- 首屏推荐内容：6 条以上
- 正片 tab：至少 6 条可浏览
- 日常 tab：至少 6 条可浏览
- 招募 tab：至少 4 条可浏览

## 六、需求招募内容规划

### 6.1 需求池分类

| 需求类型 | 建议数量 | 主要用于 |
| --- | --- | --- |
| 摄影招募 | 2~3 条 | 首页招募流、需求详情、需求相关页 |
| 模特招募 | 2~3 条 | 补足人物协作需求场景 |
| 妆造 / 毛发 / 道具需求 | 2~3 条 | 增强垂类专业度与差异化 |

### 6.2 页面承接

| 页面 / 模块 | 需要的内容 |
| --- | --- |
| 首页“招募”tab | 摄影招募 + 模特招募 + 妆造/毛发/道具需求混排 |
| 需求详情页 | 每一类至少有 1 条可点击详情的数据 |
| 我的需求 / 我的参与 | 至少预留少量归属数据，方便后续演示和自测 |

### 6.3 需求页上线最低要求

建议上线时至少保证：

- 3 类需求都存在
- 至少 1 条需求处于已关闭或已完成状态
- 至少 2 条需求有明确时间、地点、预算信息

这样可以避免需求页只像一个空壳表单入口。

## 七、活动资讯规划

### 7.1 数量建议

- 活动资讯：5~10 条

### 7.2 主要用途

| 页面 / 模块 | 作用 |
| --- | --- |
| 活动页“活动资讯”tab | 保证信息密度 |
| 首页 Banner / 推荐位 | 提供活动相关跳转承接 |
| 活动详情页 | 作为“近期活动感”支撑 |

### 7.3 内容结构建议

活动资讯建议覆盖：

- 近期漫展排期
- 城市活动预告
- 展会交通 / 入场提醒
- 报名信息汇总
- 返图征集 / 主题征集

这样活动页看起来更像持续更新的信息区，而不是单次公告页。

## 八、官方活动规划

### 8.1 数量建议

- 官方活动：1~3 条

### 8.2 主要用途

| 页面 / 模块 | 作用 |
| --- | --- |
| 活动页“官方活动”tab | 核心承接页 |
| 首页 Banner | 引导报名与活动详情点击 |
| 首页推荐内容 | 通过活动预热帖或活动返图形成联动 |

### 8.3 最低要求

建议至少保证：

- 1 条可报名的官方活动
- 若内容资源充足，可补 1 条进行中或已结束活动，增强状态层次

## 九、各页面上线最低内容标准

### 9.1 首页

- Banner：1~2 条
- 推荐流：10 条以上混合帖子
- 正片：8 条以上
- 日常：8 条以上
- 招募：6 条以上

### 9.2 活动页

- 活动资讯：5 条以上
- 官方活动：1 条以上

### 9.3 需求页 / 招募入口

- 摄影招募：2 条以上
- 模特招募：2 条以上
- 妆造 / 毛发 / 道具需求：2 条以上

### 9.4 发现页

- 至少有足够帖子内容生成一批可点击标签
- 推荐内容不应少于 10 条

## 十、推荐内容分发方式

为了避免内容虽然够数，但页面呈现仍显得单薄，建议按下面方式分发：

- 首页推荐流：正片 40% + 日常 40% + 活动返图 20%
- 首页招募流：摄影 / 模特 / 妆造类尽量均衡
- 活动页：资讯多于官方活动，保持“持续有内容”的感觉
- 发现页：优先使用正片和日常，标签覆盖 cosplay、摄影、妆造、漫展、返图、同城等

## 十一、冷启动阶段的内容优先级

### P0：上线前必须准备

- 正片作品
- 日常分享
- 活动资讯
- 至少 1 条官方活动
- 至少 3 类需求招募

### P1：建议同步准备

- 活动返图
- 官方活动预热帖
- 城市 / 同城类标签内容

### P2：上线后继续补充

- 更多专题帖
- 节日活动专题
- 返图合集与二次传播内容

## 十二、执行建议

推荐按以下顺序建设内容池：

1. 先准备首页帖子池
2. 再补需求招募池
3. 再补活动资讯池
4. 最后补官方活动与 Banner 联动内容

这样能最快避免“小程序上线像空站”的问题。

## 十三、上线前检查 Checklist

- [ ] 首页推荐流至少有 10 条可浏览内容
- [ ] 首页“正片”至少有 8 条内容
- [ ] 首页“日常”至少有 8 条内容
- [ ] 首页“招募”至少有 6 条内容
- [ ] 活动页“活动资讯”至少有 5 条内容
- [ ] 活动页“官方活动”至少有 1 条内容
- [ ] 摄影 / 模特 / 妆造毛发道具三类需求均已覆盖
- [ ] 首页、活动页、需求页都不会出现明显空站感

## 十四、可直接丢给 Trae 的 Prompt

请为“粤次元君”整理阶段 6 冷启动内容池规划。

要求：

1. 覆盖首页帖子内容、需求招募内容、活动资讯、官方活动
2. 给出每类内容的建议数量
3. 说明这些内容分别用于哪些页面和模块
4. 明确每个页面上线时至少要有多少内容
5. 输出 `content-seed-plan-v1.md`
