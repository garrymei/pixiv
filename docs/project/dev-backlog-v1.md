## 阶段 2: 前端骨架与基础页面 (Phase 2)

> **前置依赖**: 阶段 0 和 阶段 1 的所有设计文档已定稿。

| 任务编号 | 任务名称 | 输入依赖 | 输出产物 | 验收标准 | 优先级 |
|---|---|---|---|---|---|
| **FE-01** | 前端脚手架初始化 | 无 | `frontend/` 目录下的可运行项目代码 | 能够成功启动小程序预览，引入了 Design Tokens，配置好路由表 (app.json) | P0 (最高) |
| **FE-02** | 基础组件开发 | `component-inventory.md`, `design-tokens.md` | NavBar, Button, Tag, EmptyState 组件 | 组件可以在空页面中渲染并符合视觉要求 | P0 |
| **FE-03** | 业务卡片组件开发 | `FE-02`, `page-field-mapping.md` | PostCard, DemandCard, EventCard | 输入假数据(Mock)能正确渲染卡片，包含超长文本截断等规则 | P1 |
| **FE-04** | TabBar 页面 UI 搭建 | `FE-03`, `page-specs.md` | 首页、发现、发布(入口)、活动、我的 5个主页 | 页面结构完整，能够点击 Tab 互相切换，展示静态卡片列表 | P1 |
| **FE-05** | 核心二级页 UI 搭建 | `FE-02`, `page-specs.md` | 帖子详情、需求详情、活动详情页 | 页面元素完整，能从主页点击跳转并传递 ID 参数 | P1 |
| **FE-06** | 发布/表单页 UI 搭建 | `FE-02` | 发布帖子、发布需求、编辑资料页 | 表单元素齐全，包含图片上传占位区，校验逻辑占位 | P2 |

## 阶段 3: 后端核心能力与联调 (Phase 3)

> **前置依赖**: `database-design-v1.md`, `api-spec-v1.md`, `mysql-init-v1.sql`。

| 任务编号 | 任务名称 | 输入依赖 | 输出产物 | 验收标准 | 优先级 |
|---|---|---|---|---|---|
| **BE-01** | 后端项目初始化 | 无 | `backend/` 目录可运行项目 | Node.js (或所选技术栈) 启动成功，能连接 MySQL，包含基础的 Error Handler 和统一返回格式 | P0 |
| **BE-02** | 数据库连通与 ORM 配置 | `mysql-init-v1.sql` | 数据库实体/模型定义代码 | 能通过代码成功执行简单的 CRUD | P0 |
| **BE-03** | 用户鉴权模块开发 | `api-spec-v1.md` | 登录API、获取/修改资料API、JWT 中间件 | 能生成 Token，通过 Token 获取用户信息，无 Token 访问受限接口返回 401 | P0 |
| **BE-04** | 帖子(作品/日常)模块开发 | `api-spec-v1.md` | 帖子 CRUD 接口、点赞接口 | 能发布图文，获取列表时支持分页和按时间倒序，软删除生效 | P1 |
| **BE-05** | 评论模块开发 | `BE-04` | 评论 CRUD 接口 | 能对特定帖子发表评论，获取列表能关联用户头像昵称 | P1 |
| **BE-06** | 需求大厅模块开发 | `api-spec-v1.md` | 需求 CRUD 接口、申请参与接口 | 需求状态流转正常，不能重复申请同一个需求 | P1 |
| **BE-07** | 活动资讯模块开发 | `api-spec-v1.md` | 活动只读接口、报名接口 | 能按状态(进行中/已结束)返回活动，报名人数超限时拦截并报错 | P1 |
| **BE-08** | 图片上传服务 | `api-spec-v1.md` | 统一的图片上传 API | 前端调用接口可上传图片到本地或云存储，并返回可访问的 URL | P1 |
