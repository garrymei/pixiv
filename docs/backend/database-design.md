# 粤次元君 - 数据库设计初稿 (MySQL 8.0)

本文档定义了“粤次元君”MVP 阶段的核心数据库表结构。

## 全局设计规约
1. **字符集**：所有表默认使用 `utf8mb4`，排序规则使用 `utf8mb4_unicode_ci`，以完美支持 Emoji 表情。
2. **时间字段**：每张核心业务表均包含 `created_at` 和 `updated_at`，由 MySQL 自动维护。
3. **状态字段**：统一使用 `status` 字段管理数据状态。通常 `1` 为正常，`0` 为禁用/下架，`-1` 为删除（软删除）。
4. **主键设计**：统一使用无符号大整型自增 (`BIGINT UNSIGNED AUTO_INCREMENT`) 作为主键。

---

## 核心表说明

### 1. `users` (用户表)
存储小程序的注册用户信息及基础资料。
- 关键字段：`openid` (微信唯一标识), `role_type` (身份标识)。

### 2. `posts` (帖子/图文表)
存储用户发布的作品与日常动态。
- 关键字段：`post_type` (作品/日常), 冗余 `like_count` 和 `comment_count` 提升查询性能。

### 3. `post_images` (帖子图片表)
将帖子与图片解耦，支持一对多（最多9张）。

### 4. `comments` (评论表)
帖子的互动评论。
- 关键字段：`parent_id` 预留给未来二阶段的楼中楼功能，MVP阶段可默认为0。

### 5. `likes` (点赞记录表)
记录用户的点赞行为，使用多态设计 `target_type` (post/comment)，便于后期扩展。

### 6. `events` (活动/资讯表)
官方或展会的活动信息。
- 包含活动的开始/结束时间、报名截止时间等业务字段。

### 7. `event_registrations` (活动报名表)
记录用户报名活动的数据。
- `form_data_json`：JSON 字段，用于灵活存储表单数据（姓名、联系方式等）。

### 8. `demands` (协作需求表)
用户发布的约拍、求妆等协作需求。
- 关键字段：`demand_type` (角色类型), `budget_type` (互免/付费)。

### 9. `demand_applications` (需求应征表)
其他用户针对某项需求提交的接单/应征申请。

### 10. `banners` (首页/活动页轮播图表)
用于后台配置小程序的轮播图或广告位。

### 11. `tags` (标签字典表)
系统预设或用户自定义的标签。

### 12. `post_tags` (帖子-标签关联表)
实现帖子与标签的多对多关联。
